ИНСТРУКЦИЯ ПО ЗАГРУЗКЕ САЙТА НА ХОСТИНГ БЕЗ СТРАНИЦЫ СТРОИТЕЛЬСТВА
=====================================================================

✅ ВЫПОЛНЕНО:
- Удалены все ссылки на страницу "Строительство 2026" из меню всех страниц
- Удалены ссылки из десктопного меню, мобильного меню и футера

📁 ЧТО НУЖНО СДЕЛАТЬ ПРИ ЗАГРУЗКЕ:

1. ПОДГОТОВКА ФАЙЛОВ:
   - Скопируйте весь проект на хостинг
   - НЕ загружайте папку "construction" (или удалите её после загрузки)

2. СТРУКТУРА ПАПОК ДЛЯ ЗАГРУЗКИ:
   ✅ Загрузить:
   - index.html (главная страница)
   - style.css (основные стили)
   - about/ (папка "О нас")
   - design/ (папка "Проектирование")
   - news/ (папка "Новости")
   - partners/ (папка "Партнеры")
   - administrative/ (папка "Административные сооружения")
   - main_video/ (видео для главной страницы)
   - main_img/ (изображения)
   - Все остальные папки с ресурсами (img, css, js и т.д.)
   
   ❌ НЕ загружать:
   - construction/ (папка со страницей строительства)

3. СПОСОБЫ ЗАГРУЗКИ:

   СПОСОБ 1: Через FTP/FileZilla
   - Подключитесь к хостингу через FTP
   - Загрузите все файлы и папки, кроме папки "construction"
   
   СПОСОБ 2: Через панель управления хостингом
   - Загрузите архив проекта
   - Распакуйте архив на сервере
   - Удалите папку "construction" через файловый менеджер
   
   СПОСОБ 3: Через Git (если используется)
   - Добавьте в .gitignore строку: construction/
   - Сделайте коммит и пуш
   - На сервере выполните git pull

4. ПРОВЕРКА ПОСЛЕ ЗАГРУЗКИ:
   - Откройте главную страницу сайта
   - Проверьте, что в меню нет пункта "Строительство 2026"
   - Проверьте все ссылки в меню - они должны работать
   - Убедитесь, что страница construction.html недоступна (должна быть ошибка 404)

5. ВОССТАНОВЛЕНИЕ (если понадобится):
   Если в будущем захотите вернуть страницу строительства:
   - Загрузите папку "construction" на хостинг
   - Восстановите ссылки в меню (можно через Git или вручную)

⚠️ ВАЖНО:
- Папка "construction" остается в вашем локальном проекте
- Она просто не загружается на хостинг
- Все изменения сохранены в Git (если используете)

📝 ПРИМЕЧАНИЕ:
Если вы используете систему контроля версий (Git), рекомендуется создать отдельную ветку 
для продакшена или использовать .gitignore для исключения папки construction при деплое.
