ФАЙЛЫ ДЛЯ УДАЛЕНИЯ ПЕРЕД ЗАГРУЗКОЙ НА ХОСТИНГ
===============================================

📋 СПИСОК ЛИШНИХ ФАЙЛОВ:

1. config.example.js
   - Пример конфигурации, не нужен на хостинге
   - На хостинге должен быть только config.js с реальными данными

2. README.md
   - Документация для разработчиков
   - Не нужна на продакшене

3. FAVICON_INSTRUCTIONS.md
   - Инструкция по созданию favicon
   - Не нужна на хостинге

4. ИНСТРУКЦИЯ_ЗАГРУЗКА_НА_ХОСТИНГ.txt
   - Инструкция по загрузке
   - Не нужна на хостинге

5. .gitignore
   - Файл для Git
   - Не критично, но обычно не нужен на хостинге

6. construction/ (ВСЯ ПАПКА)
   - Папка со страницей строительства
   - Согласно инструкции, не должна загружаться на хостинг

⚠️ ВАЖНО: НЕ УДАЛЯЙТЕ config.js!
   - config.js должен быть на хостинге с реальными данными Telegram бота
   - Проверьте, что в config.js указаны правильные BOT_TOKEN и CHAT_ID

📁 ФАЙЛЫ, КОТОРЫЕ ДОЛЖНЫ БЫТЬ НА ХОСТИНГЕ:

✅ ОБЯЗАТЕЛЬНЫЕ:
- index.html
- style.css
- index.js
- config.js (с реальными данными!)
- favicon.svg
- site.webmanifest
- robots.txt
- sitemap.xml
- snow.css

✅ ПАПКИ:
- about/
- design/
- news/
- partners/
- administrative/
- main_img/
- main_video/
- (все остальные папки с ресурсами)

❌ НЕ ЗАГРУЖАТЬ:
- construction/ (вся папка)
- config.example.js
- README.md
- FAVICON_INSTRUCTIONS.md
- ИНСТРУКЦИЯ_ЗАГРУЗКА_НА_ХОСТИНГ.txt
- .gitignore

🔍 ПРОВЕРКА ПЕРЕД ЗАГРУЗКОЙ:

1. Убедитесь, что config.js содержит реальные данные:
   - BOT_TOKEN должен быть заполнен
   - CHAT_ID должен быть заполнен

2. Проверьте robots.txt - там есть ссылка на construction/
   (можно оставить, но лучше удалить строку Allow: /construction/)

3. Проверьте sitemap.xml - там может быть ссылка на construction/
   (нужно удалить, если страница не будет на хостинге)
